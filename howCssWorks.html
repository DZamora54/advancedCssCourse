How CSS is Parsed, Part 1: The Cascade and Specificity

The Cascade

Process of combining different stylesheets and resolving conflicts between different CSS rules and declarations, when more than one rule applies to a certain element.

  • Author declarations: the CSS that the developers write and put in the stylesheets.
  • User declarations: the CSS coming from the user. For instance when the user changes the default font size in the browser.
  • Browser (user agent) declarations: the CSS set by the browser.

The cascade combines the CSS declarations coming from these sources using importance, specificity and source order.

Importance

  1. User !important declarations
  2. Author !important declarations
  3. Author declarations
  4. User declarations
  5. Default browser declarations

Specificity

When importance is the same, specificity is used.

Specificity categories
  1. Inline styles
  2. IDs
  3. Classes, pseudo-classes, attribute
  4. Elements, pseudo-classes

Add the number of occurances of each specificity category for the selector and list these left to right: (inline, IDs, classes, elements) as (a, b, c, d). To determine the most specific selector, known as the cascaded value, compare the occurances of the first category. If there is a selector with more occurances of a compared category then that selector is the cascaded value. If all selectors have the same number of occurances of a compared category, compare the occurances of the next category.

When specificity is all the same, source order is used.

Source Order

The last declaration in the code will override all other declarations and will be applied.

Summary

  • CSS declarations marked with !important have the highest priority;
  • But, only use !important as a last resource. It's better to use specificities - more maintanable code!
  • Inline styles will always have priority over styles in external stylesheets;
  • A selector that contains 1 ID is more specific than one with 1000 classes;
  • A selector that contains 1 class is more specific than one with 1000 elements;
  • The universal selector * has no specificity value ( 0, 0, 0, 0 ) which means all other selectors have precedence over the universal selector;
  • Rely more on specificity than on the order of selectors;
  • But, rely on order when using 3rd-party stylesheets - always put your author stylesheets last.
